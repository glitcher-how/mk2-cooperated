# Архитектура репозитория

Репозиторий организован как монорепозиторий с разделением на переиспользуемые компоненты и прикладные системы.

## Состав

- `components/sdk-core` — общие базовые классы `BaseComponent` и `BaseGateway`.
- `broker` — шина сообщений для локальной разработки и тестирования.
- `systems/operator` — система оператора БАС.

## Правила

1. Исходный код хранится только в `src/`.
2. Документация сервисов хранится только в `docs/`.
3. Сгенерированные артефакты в Git не коммитятся.
4. Диаграммы последовательности описываются на PlantUML и лежат в `docs/diagrams/`.
5. Прочие архитектурные схемы хранятся в формате Draw.io.
